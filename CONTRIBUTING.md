# Contributing to TRS Alliance

Thank you for your interest in contributing to TRS Alliance! We welcome contributions from the community.

## Legal Requirements

**By contributing, you agree:**
- All contributions must be your original work
- You have the right to assign the copyright for your work
- Your contributions are made in good faith for legitimate purposes only
- You comply with all applicable laws and regulations

## Code of Conduct

Please read our [Code of Conduct](CODE_OF_CONDUCT.md) before contributing.

## How to Contribute

### Reporting Issues
- Use the issue templates provided
- Search existing issues before creating new ones
- Provide detailed, reproducible bug reports

### Submitting Changes
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request

### Pull Request Process
1. Ensure tests pass and code follows our standards
2. Update documentation as needed
3. Request review from maintainers
4. Address review feedback promptly

## Style Guides

### Code Style
- Follow existing code patterns
- Write clear, readable code
- Add comments for complex logic

### Commit Messages
- Use present tense ("Add feature" not "Added feature")
- Use imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit first line to 72 characters

## Security

- Do not add hardcoded secrets or credentials
- Report security vulnerabilities via [SECURITY.md](SECURITY.md)
- All code will be security reviewed before merging
